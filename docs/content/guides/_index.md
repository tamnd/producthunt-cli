Each guide is built around a job rather than a command.
They assume you have run the [quick start](/getting-started/quick-start/).

- [Set up the API plane](/guides/credentials/) gets free credentials and turns on the reliable GraphQL reads.
- [Script and pipe ph](/guides/scripting/) shapes records for `jq`, spreadsheets, and a local store, and serves them over HTTP.
- [Add a command](/guides/adding-a-command/) models a real Product Hunt record and turns it into a command, a route, and a tool at once.
- [Resource URIs](/guides/resource-uris/) uses ph as a `database/sql`-style driver so a host can address Product Hunt as `producthunt://` URIs.
