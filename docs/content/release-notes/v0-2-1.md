A maintenance release that lines the docs up with what `ph` actually does and makes one exit code honest.
No command, flag, or record shape changed, so an upgrade is a drop-in replacement for v0.2.0.

## A network error now exits 8

A connection refused, a timeout, or a 5xx that never recovered after retries used to fall through to the generic exit 1, even though the reference promised exit 8 for a network error.
`ph` now returns exit 8 for a transport failure that survives every retry, so a script can tell a dead network apart from any other failure.

## Documentation corrected

A pass over the guides and the reference fixed several places where the prose ran ahead of the code:

- `--template` fields are the lower-case JSON keys, the same names `-o json` prints, so the examples now read `{{.name}}` rather than `{{.Name}}`. The capitalised form rendered an empty value.
- There is no per-flag environment fallback. The configuration page now lists only the variables `ph` reads: the `PRODUCTHUNT_` credentials, `PH_DATA_DIR`, `XDG_DATA_HOME`, and `NO_COLOR`.
- The build stores records in SQLite, so `--db` takes a file path. The `postgres://` examples are gone.
- The default delay between requests is two seconds, so the polite-run advice now suggests `--rate 5s` and the reference lists the real defaults.

## A clearer rate-limit message

The rate-limit error pointed at a `--delay` flag that does not exist.
It now names the real flag, `--rate`.
