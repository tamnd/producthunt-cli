`ph` is rebuilt on the [any-cli/kit](https://github.com/tamnd/any-cli) framework. It now reads two planes that share one numeric post id and one record shape, and every operation is a CLI command, an HTTP route, an MCP tool, and a `producthunt://` URI at once.

This replaces the v0.1.0 command surface. The old `ph today` command is gone; use `ph feed` or `ph posts` instead.

## Two planes, one record

| Plane | Reads | Turns on |
|---|---|---|
| web (default) | the keyless Atom feed at `www.producthunt.com/feed` | always; the one anonymous surface behind Cloudflare |
| api | `api.producthunt.com` over GraphQL | when `PRODUCTHUNT_TOKEN`, or `PRODUCTHUNT_CLIENT_ID` and `PRODUCTHUNT_CLIENT_SECRET`, are set in the environment |

A feed read and an API read address the same record, so a pasted link and a feed entry resolve to one id. With client id and secret set, `ph` mints a short-lived access token with the OAuth client-credentials grant on first use and reuses it for the run. Credentials are read from the environment only, never a flag. Pick a plane with `--plane web|api|auto` (default `auto`).

## Commands

| Command | Plane | What it does |
|---|---|---|
| `ph feed` | web | the keyless Atom feed of recent posts |
| `ph posts` | web or api | the post stream (the feed for the newest case, the API when keyed or filtered) |
| `ph post <ref>` | api or web | one post in full (API by id or slug, the feed when keyless and recent) |
| `ph comments <ref>` | api | a post's comment thread |
| `ph topics` | api | list topics |
| `ph topic <ref>` | api | one topic with its top posts |
| `ph collections` | api | list collections |
| `ph collection <ref>` | api | one collection with its posts |
| `ph user <ref>` | api | one user with the posts they made |
| `ph ref id <ref>` | offline | resolve any URL, Atom id, `@username`, slug, or id to its (kind, id) |
| `ph ref url <kind> <id>` | offline | build the addressable URL for a (kind, id) |

The list-shaping flags `--order`, `--topic`, `--featured`, `--after`, and `--before` apply to the post stream on the API plane. The topic, collection, user, and comment surfaces are API only and report exit 4 with a clear remedy when no credentials are set.

## Output and serving

Every command shares one output contract: `-o table|markdown|list|json|jsonl|csv|tsv|url|raw`, `--fields` to pick columns, `--template` for a custom line, and `-n` to limit. The default adapts to where output goes, a table on a terminal and JSONL in a pipe. The same operations serve over HTTP with `ph serve` and to agents over MCP with `ph mcp`.

## Resource-URI driver

The `producthunt` package registers a `producthunt` domain the way a program registers a `database/sql` driver. A host like [ant](https://github.com/tamnd/ant) enables it with one blank import and dereferences `producthunt://` URIs. Records carry link edges (a post out to its topics, makers, hunter, and comment thread, and back), so a host can walk the graph from the keyless feed. See [resource URIs](/guides/resource-uris/).

## Exit codes

| Code | Meaning |
|---|---|
| 0 | ok |
| 2 | usage |
| 3 | no results |
| 4 | needs credentials, or a walled web page |
| 5 | rate limited or query too complex |
| 6 | not found |
| 8 | network error |

## Install

```bash
go install github.com/tamnd/producthunt-cli/cmd/ph@v0.2.0
```

Or download a prebuilt binary from the [v0.2.0 release](https://github.com/tamnd/producthunt-cli/releases/tag/v0.2.0).

### Docker

```bash
docker pull ghcr.io/tamnd/ph:0.2.0
docker run --rm ghcr.io/tamnd/ph:0.2.0 feed -n 5
```
